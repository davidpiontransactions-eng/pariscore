# pm-execution

Compétences en exécution et gestion de produit : PRDs, OKRs, roadmaps, sprints, pre-mortems, cartes des parties prenantes, user stories, frameworks de priorisation, et plus encore.

## Skills (16)

- **brainstorm-okrs** — Générer des OKRs d'équipe alignés sur les objectifs de l'entreprise.
- **create-prd** — Rédiger un document de spécification produit (PRD) avec un template complet en 8 sections.
- **dummy-dataset** — Générer des jeux de données réalistes pour les tests (CSV, JSON, SQL, script Python).
- **job-stories** — Créer des job stories au format « Quand [situation], je veux [motivation], afin de [résultat] ».
- **outcome-roadmap** — Transformer une roadmap orientée fonctionnalités en roadmap orientée résultats.
- **pre-mortem** — Analyser les risques d'un PRD avant le lancement.
- **prioritization-frameworks** — Guide de référence pour 9 frameworks de priorisation.
- **release-notes** — Générer des notes de version orientées utilisateur.
- **retro** — Faciliter une rétrospective de sprint structurée.
- **sprint-plan** — Planifier un sprint avec estimation de capacité et identification des risques.
- **stakeholder-map** — Cartographier les parties prenantes avec une grille Pouvoir × Intérêt.
- **strategy-red-team** — Stress-test d'un PRD ou d'une stratégie en attaquant ses hypothèses critiques.
- **summarize-meeting** — Résumer un transcript de réunion en notes structurées.
- **test-scenarios** — Créer des scénarios de test complets à partir de user stories.
- **user-stories** — Rédiger des user stories selon les critères 3C et INVEST.
- **wwas** — Créer des éléments de backlog au format Why-What-Acceptance.

## Commandes (11)

- `/pm-execution:generate-data` — Générer des jeux de données réalistes pour les tests.
- `/pm-execution:meeting-notes` — Résumer un transcript de réunion en notes structurées.
- `/pm-execution:plan-okrs` — Générer des OKRs d'équipe avec objectifs qualitatifs et KRs mesurables.
- `/pm-execution:pre-mortem` — Analyse des risques avant lancement.
- `/pm-execution:red-team-prd` — Stress-test d'un PRD ou d'une stratégie.
- `/pm-execution:sprint` — Cycle de sprint complet (planification, rétro, notes de version).
- `/pm-execution:stakeholder-map` — Cartographie des parties prenantes et plan de communication.
- `/pm-execution:test-scenarios` — Générer des scénarios de test à partir de spécifications.
- `/pm-execution:transform-roadmap` — Convertir une roadmap fonctionnelle en roadmap orientée résultats.
- `/pm-execution:write-prd` — Rédiger un document de spécification produit complet.
- `/pm-execution:write-stories` — Décomposer une fonctionnalité en éléments de backlog.
