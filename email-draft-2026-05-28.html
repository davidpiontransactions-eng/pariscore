TENNIS DATA PIPELINE — v12.66

Bonjour,

Ce soir, l’onglet Tennis de PariScore reçoit une mise à jour majeure de son pipeline de données statistiques. Voici le détail des évolutions déployées ce 28 mai 2026.

Fix #4 — Source Elo

Switch source legacy → union

Le calculateur Elo lisait exclusivement la table Sackmann (sync désactivé depuis la purge CC-BY-NC-SA). Toutes les stats étaient figées. La source par défaut est maintenant union : Sackmann legacy + tennis_matches_internal (ETL propriétaire BSD/ESPN).

→ Elo, surface rank, form, powerscore : valeurs fraîches chaque jour.

Fix #1 — Cron quotidien

Recomputation Elo automatique à 02h10

computeTennisElo() ne tournait qu’au boot, puis plus jamais. Un cron quotidien rebuild désormais la table après l’ETL nocturne (02h00).

→ Ratings Elo automatiquement à jour chaque nuit.

Fix #2 — Live WElo

Mise à jour Elo en direct depuis les matchs live

Les matchs terminés détectés par pollTennisLive() (toutes les 30s) alimentent désormais tennis_players_elo automatiquement. Résolution player_id par nom si absent du flux BSD/ESPN.

→ WElo suit le circuit en temps réel, plus seulement au boot.

Fix #3 — SPS Pipeline

Fallback player_id pour le cron SPS

L’endpoint /api/v1/tennis/upcoming exigeait un player_id que BSD ne fournit pas toujours. Résolution par nom dans tennis_players_elo en fallback.

→ Plus de matchs traités par le cron SPS → chips SPS visibles.

Calendrier des crons

Toutes les 30spollTennisLive + WElo auto-update
02h00 ParisETL tennis_matches_internal (BSD/ESPN)
02h05 ParisSPS updater (player_surface_scores)
02h10 ParisRecompute tennis_elo (union source)
06h00 ParisTex daily refresh + odds snapshots

Le déploiement a été effectué ce soir à 22h30. Les premiers résultats (Elo frais, SPS, WElo live) seront visibles dès le cycle cron de 02h10.

À bientôt sur pariscore.fr.
