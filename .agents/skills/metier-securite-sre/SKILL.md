# 🛡️ Métier — Sécurité & SRE

> **Rôle** : Protéger le système, gérer les incidents, assurer la fiabilité et la disponibilité.

## Skills disponibles

| # | Skill | Nom d'appel | Fonction |
|---|-------|------------|----------|
| 1 | Security Architect | `/agency-security-architect` | Threat modeling STRIDE, defense-in-depth, OWASP checklist 30+ points — adapté PariScore (Node.js REST, SQLite, Render.com, .env) |
| 2 | SRE | `/agency-sre` | SLOs/SLIs, error budgets, golden signals, observabilité. SLOs recommandés : dispo 99.5%, p95 < 500ms |
| 3 | Incident Commander | `/agency-incident-commander` | Commandement incident SEV1-SEV4, 3 runbooks pré-construits (Render crash, API down, DB corruption), post-mortem template |
| 4 | Playwright MCP | `/playwright-mcp` | Debug visuel incident — capture screenshot état bug, vérifier UI après hotfix |

## Quand utiliser ce métier

- Audit sécurité ou feature sensible → `/agency-security-architect`
- Définir/revoir les SLOs → `/agency-sre`
- Incident production → `/agency-incident-commander`
- Capture screenshot pour post-mortem / vérifier UI post-hotfix → `/playwright-mcp`
