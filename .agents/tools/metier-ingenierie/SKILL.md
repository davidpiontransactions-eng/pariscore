# 🔧 Métier — Ingénierie & Coding

> **Rôle** : Écrire, réviser, tester et maintenir le code PariScore. Accompagnement au développement quotidien.

## Skills disponibles

| # | Skill | Nom d'appel | Fonction |
|---|-------|------------|----------|
| 1 | Code Reviewer | `/agency-code-reviewer` | Review structuré 🔴 blocker / 🟡 suggestion / 💭 nit — adapté PariScore (ES5, SQLite, STRATEGIES sync) |
| 2 | Caveman Review | `/caveman-review` | Review ultra-compressé, 1 ligne par comment |
| 3 | Caveman Commit | `/caveman-commit` | Commits compressés au format Conventional Commits |
| 4 | Caveman (mode) | `/caveman` | Mode communication ultra-compressé (~75% tokens en moins) |
| 5 | Caveman Compress | `/caveman-compress` | Compresse fichiers mémoire en format caveman |
| 6 | Caveman Help | `/caveman-help` | Quick-reference de tous les modes caveman |
| 7 | Caveman Stats | `/caveman-stats` | Usage réel de tokens de la session |
| 8 | Cavecrew | `/cavecrew` | Délégation multi-agent compressée (investigator/builder/reviewer) |
| 9 | Playwright MCP | `/playwright-mcp` | Tests navigateur intégrés — vérifier UI, capture screenshot, debug visuel pendant le dev |

## Quand utiliser ce métier

- Vous écrivez du code → `/agency-code-reviewer` pour review
- Vous faites un commit → `/caveman-commit`
- Le contexte est trop gros → `/caveman` pour switcher en mode compressé
- Vous voulez monitorer → `/caveman-stats`
- Vous voulez tester visuellement dans le navigateur → `/playwright-mcp`
