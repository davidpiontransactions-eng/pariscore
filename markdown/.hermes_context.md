## Pourquoi V73.0 est LA solution

Les versions V70-V72 essayaient de **corriger** le scroll après avoir **détruit** le DOM. C'est impossible à rattraper de manière fiable à cause du timing async (buteurs, predictions) et de l'ancrage scroll natif du navigateur.

V73.0 s'inspire du standard industriel (FlashScore/SofaScore/Bet365) : **patch différentiel**. On ne détruit jamais les `<tr>`, on met à jour uniquement les valeurs modifiées via `data-*` selectors.

## Architecture

```
patchLiveMatches(matches)
  ├─ Pour chaque match live : querySelector('[data-live="ID"]')
  ├─ badge.innerHTML = nouveau score/minute
  ├─ intensityBar.style.width/color = nouvelle intensité
  ├─ _prevLiveIds pour détecter matchs terminés
  └─ ZÉRO destruction DOM → ZÉRO scroll jump

liveUpdateProtocol(newMatches, meta)
  ├─ LIVE tab → patchLiveMatches (95% des cas, pas de rebuild)
  │              └─ si structural change → renderMatches (5%, acceptable)
  └─ Autres tabs → renderMatches (5min interval, scroll pas critique)
```
