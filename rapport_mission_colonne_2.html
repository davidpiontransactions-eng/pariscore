4. Architecture réseau

4.1 Existant

  • Données 100 % locales SQLite (tennis_elo + tennis_matches).
  • getTennisSurfStats(name,tour,surface) : rang surface + forme 90j même surface.
  • Cache mémoire TTL 30 min ; injecté dans payload /api/v1/tennis/value-bets.

4.2 Évolution proposée

GET /api/v1/tennis/player-stats?name=<...>&tour=ATP|WTA&surface=Clay|Hard|Grass|Carpet
→ { rk, total, form:["W","L","W"], cached_at, ttl_h:12 }

Cache JSON disque cache/tennis_player_stats.json, validité 12h. Lecture seule, zéro scrape runtime → anti-ban IP garanti.
