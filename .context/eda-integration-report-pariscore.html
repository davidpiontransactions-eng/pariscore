Usage admin — exemples curl
1. Audit qualité table player_surface_scores (fg-data-profiling)
curl -H "Authorization: Bearer <ADMIN_JWT>" \
  "http://localhost:3000/api/v1/admin/eda/profile?table=player_surface_scores"
# → {"ok":true,"rows":112,"cols":8,"url":"/eda/profile/player_surface_scores"}
# Ouvrir: http://localhost:3000/eda/profile/player_surface_scores
2. GUI interactif D-Tale sur archive_matches
curl -H "Authorization: Bearer <ADMIN_JWT>" \
  "http://localhost:3000/api/v1/admin/eda/dtale?table=archive_matches&port=40000"
# → {"ok":true,"url":"http://localhost:40000/dtale/main/1","rows":32918}
3. Chat PandasAI (nécessite OPENAI_API_KEY dans .env)
curl -X POST -H "Authorization: Bearer <ADMIN_JWT>" \
  -H "Content-Type: application/json" \
  -d '{"table":"player_surface_scores","question":"Quel joueur a le meilleur SPS sur clay?"}' \
  "http://localhost:3000/api/v1/admin/eda/chat"
# → {"ok":true,"answer":"Jannik Sinner (SPS clay: 92.4)","table":"player_surface_scores"}

Insights — apport concret PariScore
🎯
player_surface_scores auditée en 1 click

GET /api/v1/admin/eda/profile?table=player_surface_scores → rapport HTML complet : distribution SPS par surface, missing values, corrélations player_id/score. Détecte instantanément si le cron SPS a mal écrit des données.

📈
D-Tale remplace DBeaver pour l'exploration live de archive_matches

32 918 records navigables avec filtres pandas, charts Plotly inline, tri/groupby en GUI. Repère en 5 min les distributions d'Elo anormales ou les dates de match mal parsées — sans écrire une seule ligne SQL.

💬
PandasAI : "Quels joueurs SPS clay > 80 avec moins de 10 matchs?"

POST /api/v1/admin/eda/chat → PandasAI traduit la question en pandas code, exécute sur SQLite, retourne la réponse en JSON. Idéal pour valider les seuils de confidence_full avant de changer le flag SPS dans le frontend.

🔒
Toutes routes protégées JWT admin — 0 exposition publique

Chaque route EDA valide getAuthUser(req).role === 'admin'. Les rapports HTML générés sont servis sur /eda/profile/:table (non indexés par robots.txt). EDA_PYTHON_BIN env override pour VPS custom path.

⚡
Économie estimée : 24-36h de R&D/mois

Sur 12 sessions d'exploration data par mois (calibration Poisson, nouveaux signals, backtest debug), le gain de 75-80% sur le temps d'EDA représente 24-36h économisées — soit 2-3 jours dev équivalent récupérés.

Méthodologie & Architecture technique
  • Venv Python 3.11 : .venv-data/ créé avec uv — isolé de system Python 3.14 incompatible
  • Packages résolus : fg-data-profiling 4.19.1 (fork ydata-profiling), D-Tale 3.22.0, PandasAI 2.3.2
  • Conflits résolus : ABI numpy/pandas mismatch → numpy pinned 2.3.5 ; setuptools 70.x pour pkg_resources ; fg-data-profiling remplace ydata-profiling (visions 0.8.1 compatible sans numba.generated_jit)
  • Integration pattern : Node.js → child_process.spawn(_EDA_PY, [script, ...args]) → JSON stdout → jsonResponse(res, 200)
  • Script pivot : tools/pariscore-eda.py — CLI argparse multi-mode (profile/dtale/chat/tables) · output JSON
  • Sécurité : table name sanitized .replace(/[^a-z0-9_]/gi, '') · auth JWT admin sur toutes routes · question LLM tronquée 500 chars
  • Syntaxe validée : node --check server.js → SYNTAX OK
  • EDA_PYTHON_BIN : env var override — .venv-data/Scripts/python.exe Windows · python3 VPS Linux
