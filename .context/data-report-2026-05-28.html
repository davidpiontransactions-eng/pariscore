Bugs détectés et corrigés

BUG-1 CRITICAL — Auth manquante /eda/profile/:table

Route servait rapports HTML (données DB internes) sans JWT. Fix : getAuthUser(req) → 403 si non-admin.

BUG-2 HIGH — JSON parse crash sur warnings Python

JSON.parse(raw) crashait si Python émettait FutureWarning avant le JSON. Fix : _edaParseOut() extrait la dernière ligne {.

BUG-3 HIGH — D-Tale subprocess mort au spawn

Flask D-Tale mourait dès fin du script Python. Fix : Node spawn detached+stdio:ignore+unref() ; Python print→flush→sleep(7200).

BUG-4 MED — Linux venv path Windows

Default Linux utilisait Scripts/python.exe. Fix : ternaire OS → bin/python sur Linux.

BUG-5 MED — Table vide après sanitize → SQL crash

"../secret" → sanitize → "" → SELECT * FROM crash. Fix : _edaTable() retourne null → HTTP 400.

BUG-6 LOW — Port D-Tale non validé

Port 22 accepté → conflit système. Fix : clamp max(1024, min(65534, port)).

W1 — PandasAI retourne HTTP 200 + ok:false sans clé

Sans OPENAI_API_KEY, retourne 200 au lieu de 503. Non-bloquant. Roadmap : HTTP 503 explicite.

W4 — D-Tale port 40000 non protégé firewall

D-Tale sans auth si VPS expose port 40000. Recommandation : ufw deny 40000 post-deploy.

Déploiement VPS — commandes

OVH Ubuntu — séquence install

# 1. Depuis /home/ubuntu/pariscore
bash tools/install-eda-vps.sh

# 2. Ajouter à .env VPS
EDA_PYTHON_BIN=/home/ubuntu/pariscore/.venv-data/bin/python

# 3. Protéger D-Tale port
sudo ufw deny 40000

# 4. Restart
pm2 restart pariscore

# 5. Test route tables
curl -H "Authorization: Bearer <admin_jwt>" https://domain.com/api/v1/admin/eda/tables

# 6. D-Tale via tunnel SSH
ssh -L 40000:localhost:40000 ubuntu@vps-ip
curl -H "Authorization: Bearer <admin_jwt>" "https://domain.com/api/v1/admin/eda/dtale?table=archive_matches"
# → http://localhost:40000/dtale/main/1
